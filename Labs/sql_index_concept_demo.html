Why is the index so much faster?

The index is a B-tree — a sorted, branching structure. At each level it eliminates roughly half the data with one comparison.

  • 16 rows: full scan up to 16 steps vs index ~4 steps
  • 1,000 rows: 1,000 steps vs ~10 steps
  • 1 million rows: 1,000,000 steps vs ~20 steps

Index lookup grows logarithmically, table scan grows linearly. That's why indexes feel like magic on big tables.

Create the index in SQLite
CREATE INDEX idx_users_email ON users(email);

-- Verify SQLite uses it:
EXPLAIN QUERY PLAN
SELECT * FROM users WHERE email = 'kim@x.com';
-- → SEARCH users USING INDEX idx_users_email
